Lab Notes

This repository hosts the Lab Notes website and runs the journal’s public editorial workflow.

All submissions and development work take place through GitHub Issues.

Issue Types

There are two kinds of issues:

article — submissions and peer review

website/infrastructure issues — bugs, enhancements, documentation

If an issue does not have the article label, it is considered website-related.

Article Workflow

Every article issue must have:

article

exactly one status label

Status labels:

submitted

under review

revision requested

accepted

published

rejected

When a manuscript advances, the previous status label is removed and replaced.

The issue thread serves as the public record of review and editorial decision.

Filtering

View all submissions:

is:issue label:article

Under review:

is:issue label:article label:"under review"

Published:

is:issue label:article label:published

Lab Notes operates through transparent, lightweight infrastructure.
GitHub Issues function as both editorial workspace and public archive.
